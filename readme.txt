1. Логика работы: создание файла со скриптом -> исполнение файла со скриптом (сохранение результатов во временный файл) -> просмотр результатов -> предложение пользователю удалить временный файл
2. Важно: Временный файл должен быть только один -> удаляться при перезапуске скрипта
3. Обоснование: целевым пользователям может быть важно на каждом из этапов анализа просматривать/модифицировать полученный результат, следовательно, разумно делать обмен данными именно через временный файл, а не через оперативную память (что, конечно, было бы быстрее и эффективнее)
4. Скрипт делится на 3 этапа:
	а) загрузка данных - приведение указанного пользователем файла к указанному виду (Pandas DataFrame/Numpy ndarray)
	б) собственно проведение анализа (кластеризация) - с возможностью выбора различных параметров
	в) сохранение и обобщение результатов (например, построение графиков)
